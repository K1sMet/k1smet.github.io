rocketmq双主部署失败

发表于2021-05-19|更新于2021-05-19
|字数总计:1.6k|阅读时长:5分钟|阅读量:

rocketmq-4.7.1获取2m-noslave中broker-a.properties配置失败

解决方法：

  1. 在/rocketmq-4.7.1/conf目录下配置broker.conf文件（内容参考broker-a.properties文件）
  2. 直接运行去bin目录下运行 mqbroker -c ../conf/broker.conf &

RocketMQ集群部署模式

  • 单master模式

    也就是只有一个 master 节点，如果master节点挂掉了，会导致整个服务不可用，线上不宜使用，适合个人学习使用

  • 多master模式

    多个 master 节点组成集群，单个 master 节点宕机或者重启对应用没有影响。
    优点：所有模式中性能最高
    缺点：单个 master 节点宕机期间，未被消费的消息在节点恢复之前不可用，消息的实时性就受到影响。
    注意：使用同步刷盘可以保证消息不丢失，同时 Topic 相对应的 queue 应该分布在集群中各个节点，而不是只在某各节点上，否则，该节点宕机会对订阅该 topic 的应用造成影响。

  • 多master 多slave异步复制模式

    在多 master 模式的基础上，每个 master 节点都有至少一个对应的 slave。master
    节点可读可写，但是 slave 只能读不能写，类似于 mysql 的主备模式。
    优点： 在 master 宕机时，消费者可以从 slave 读取消息，消息的实时性不会受影响，性能几乎和多 master 一样。
    缺点：使用异步复制的同步方式有可能会有消息丢失的问题。

  • 多master 多slave同步双写模式

    同多 master 多 slave 异步复制模式类似，区别在于 master 和 slave 之间的数据同步方式。
    优点：同步双写的同步模式能保证数据不丢失。
    缺点：发送单个消息 RT 会略长，性能相比异步复制低10%左右。
    刷盘策略：同步刷盘和异步刷盘（指的是节点自身数据是同步还是异步存储）
    同步方式：同步双写和异步复制（指的一组 master 和 slave 之间数据的同步）
    注意：要保证数据可靠，需采用同步刷盘和同步双写的方式，但性能会较其他方式低。

RocketMQ双主模式实现

考虑到具体项目情况，暂时采取双主的模式来实现消息中间件处理

1.下载rocketmq

地址： https://github.com/apache/rocketmq/archive/refs/tags/rocketmq-all-4.7.1.tar.gz（个人比较喜欢用最新的版本，但不推荐4.8版本，有个坑忘记啥了）

2.解压

tar -zxvf rocketmq-rocketmq-all-4.8.0.tar.gz

3.修改rocketmq-4.7.1/conf/broker.conf配置文件

A服务器：192.168.1.11
B服务器：192.168.1.12
不同服务器【主机名-brokerName】、【brokerIP1】需修改

# Broker所属哪个集群，默认【DefaultCluster】
brokerClusterName=rocketmq-cluster
# 本机主机名（B服务器则为broker-b）
brokerName=broker-a 
# BrokerId，必须是大等于0的整数，0表示Master，>0表示Slave，一个Master可以挂多个Slave，Master与Slave通过BrokerName来配对，默认【0】
brokerIP1=192.168.1.11
brokerId=0
# Name Server地址
namesrvAddr=192.168.1.11:9876;192.168.1.12:9876;
# 在发送消息时，自动创建服务器不存在的Topic，默认创建的队列数
defaultTopicQueueNums=4
# 是否允许Broker自动创建Topic，建议线下开启，线上关闭，默认【true】
autoCreateTopicEnable=true
# 是否允许Broker自动创建订阅组，建议线下开启，线上关闭，默认【true】
autoCreateSubscriptionGroup=true
# Broker对外服务的监听端口,默认【10911】
listenPort=10911
# 删除文件时间点，默认凌晨4点
deleteWhen=04
# 文件保留时间，默认48小时
fileReservedTime=48
# 单个conmmitlog文件大小默认1GB
mapedFileSizeCommitLog=1073741824
# 单个consumequeue文件大小默认30W条，根据业务调整
mapedFileSizeConsumeQueue=300000
# 销毁MappedFile被拒绝的最大存活时间，默认120s
# destroyMapedFileIntervalForcibly=120000
# 重试删除文件间隔
# redeleteHangedFileInterval=120000
# commitlog目录所在分区的最大使用比例，如果commitlog目录所在的分区使用比例大于该值，则触发过期文件删除
diskMaxUsedSpaceRatio=88
# 存储路径
storePathRootDir=/rocketmq-4.7.1/store
# commitLog 存储路径
storePathCommitLog=/rocketmq-4.7.1/store/commitlog
# 消息队列存储路径
storePathConsumeQueue=/rocketmq-4.7.1/store/consumequeue
# 消息索引存储路径
storePathIndex=/rocketmq-4.7.1/store/index
# checkpoint 文件存储路径
storePathCheckpoint=/rocketmq-4.7.1/store/checkpoint
# abort 文件存储路径
abortFile=/rocketmq-4.7.1/store/abort
# 限制消息大小
maxMessageSize=65536
# 一次刷盘至少需要脏页的数量，针对commitlog文件
# flushCommitLogLeastPages=4
# 一次刷盘至少需要脏页的数量,默认2页,针对 Consume文件
# flushConsumeQueueLeastPages=2
# commitlog两次刷盘的最大间隔,如果超过该间隔,将fushCommitLogLeastPages要求直接执行刷盘操作
# flushCommitLogThoroughInterval=10000
# Consume两次刷盘的最大间隔,如果超过该间隔,将忽略
# flushConsumeQueueThoroughInterval=60000
# Broker 的角色
# ASYNC_MASTER 异步复制Master
# SYNC_MASTER 同步双写Master
# SLAVE
brokerRole=ASYNC_MASTER
# 刷盘方式
# ASYNC_FLUSH 异步刷盘
# SYNC_FLUSH 同部刷盘
flushDiskType=ASYNC_FLUSH
# checkTransactionMessageEnable=false
# 服务端处理消息发送线程池数量
# sendMessageThreadPoolNums=128
# 服务端处理消息拉取线程池线程数量 默认为16加上当前操作系统CPU核数的两倍
# pullMessageThreadPoolNums=128

4.修改rocketmq文件存储位置

mkdir /usr/local/rocketmq-4.7.1/store
mkdir /usr/local/rocketmq-4.7.1/store/commitlog
mkdir /usr/local/rocketmq-4.7.1/store/consumequeue
mkdir /usr/local/rocketmq-4.7.1/store/index

5.修改RocketMQ JVM启动参数

在/rocketmq-4.7.1/bin文件夹内输入vim runbroker.sh / vim runserver.sh
根据配置修改：
JAVA_OPT=”${JAVA_OPT} -server -Xms1g -Xmx1g -Xmn512m”
JAVA_OPT=”${JAVA_OPT} -XX:MaxDirectMemorySize=1g”

6.启动rocketmq

rocketmq-4.7.1/bin目录下

  • 启动mqnamesrv
    nohup sh mqnamesrv &（没有权限则执行：chmod 777 ./mqnamesrv）
    可以查看rocketmq-4.7.1/logs/rocketmqlogs/namesrv.log日志文件看是否启动成功：The Name Server boot success 代表成功
  • 启动broker
    nohup sh mqbroker -c ../conf/broker.conf >/dev/null 2>&1 &
    可以查看rocketmq/logs/rocketmqlogs/broker.log 出现****:9876 OK两个地址 代表成功

7.关闭rocketmq

/rocketmq-4.7.1/bin目录下：mqshutdown broker、mqshutdown namesrv分别关闭

8.其他

  • 需安装jdk
  • 两台服务器需互通，开放端口9876、10911
    添加指定需要开放的端口：
    firewall-cmd –add-port=9876/tcp –permanent
    firewall-cmd –add-port=10911/tcp –permanent
    重载入添加的端口：
    firewall-cmd –reload
    查询指定端口是否开启成功：
    firewall-cmd –query-port=9876/tcp
    查看开放的端口
    firewall-cmd –zone=public –list-ports
